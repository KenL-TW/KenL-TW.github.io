## 關於技術

**Q: React 和 Vue 哪個更好？**  
A: 各有優勢。React 生態龐大，Vue 學習曲線平緩。選擇取決於團隊偏好與項目規模。

**Q: 應該學習 TypeScript 嗎？**  
A: 強烈建議。TypeScript 提供類型安全，減少運行時錯誤，特別是大型應用。

**Q: 如何選擇資料庫（SQL vs NoSQL）？**  
A: 
- **SQL (PostgreSQL)**：結構化資料、複雜查詢、事務一致性
- **NoSQL (MongoDB)**：靈活 Schema、水平擴展、高並發

**Q: DevOps 的學習順序是什麼？**  
A: 
1. Git & GitHub / GitLab
2. Docker 基礎容器化
3. CI/CD 流程 (GitHub Actions)
4. Kubernetes 編排 (選擇性進階)
5. 雲端平台 (AWS / Azure)

## 關於部署

**Q: 小型專案應該部署在哪裡？**  
A: 推薦 **Vercel**（前端）/ **Render**（後端）/ **Railway**，免費方案足以應對。

**Q: 如何實現高可用性（99.9% uptime）？**  
A:
- 使用負載均衡器 (Load Balancer)
- 多區域部署 (Multi-region)
- 自動故障轉移
- 監控告警系統
- 定期備份與災難恢復計劃

**Q: 應該使用 Serverless 還是容器化？**  
A:
- **Serverless (AWS Lambda)**：短任務、事件驅動、成本低
- **容器 (Docker + K8s)**：長時間運行、複雜應用、控制力強

## 關於性能

**Q: 如何優化 Web 應用的載入速度？**  
A:
- 程式碼分割 (Code Splitting)
- 圖片優化與懶加載 (Lazy Loading)
- CDN 加速靜態資源
- 快取策略 (Browser Cache / Redis)
- 資料庫查詢優化

**Q: 什麼時候應該使用緩存？**  
A: 當頻繁訪問相同資料且更新不頻繁時。常見於：使用者信息、商品列表、靜態頁面。
